# 디자인 피드백 레벨 기준 (v1, 개인용)

> 2026-07-22 작성. PO가 만든 프로토타입에 디자인 피드백을 줄 때, 어디서부터 어디까지 볼지 판정하는 기준.
> 일단 디어 개인 기준으로 운영. 팀 공통 규칙으로 확장 시 "L3 = 리뷰가 아니라 협업 전환" 부분을 PO 조직과 합의 필요.

---

## 1. 판정: 4가지 리스크 질문

레벨을 정하기 전에 아래 4개를 확인한다. 고객단/어드민, 트래픽은 대리 지표일 뿐이고, 실제 기준은 리스크.

| 축 | 질문 | 높음의 예 |
|---|---|---|
| 노출 | 누가 보는가? | 고객 > 파트너/판매자 > 사내 어드민 |
| 규모 | 얼마나 많이, 자주 쓰는가? | 홈·PDP급 트래픽, 매일 쓰는 운영 툴 |
| 되돌리기 비용 | 잘못되면 무슨 일이 생기고, 고치기 쉬운가? | 결제·할인·정산·데이터 삭제, 새 패턴 선례 |
| 수명 | 일회성인가 계속 남는가? | 지속 기능 > 한시 이벤트 > 내부 실험 |

주의: 어드민이라도 새 인터랙션 패턴을 처음 만드는 화면이면 이후 어드민 전체의 선례가 되므로 레벨을 올린다.

축의 역할 분리: 위 4축은 **얼마나 깊게 볼지(레벨)** 를 정하고, 오디언스(§3)는 그 레벨 안에서 **무엇에 힘줄지(관점)** 를 정한다.

## 2. 피드백 3레벨

| | L1. 안전선 체크 | L2. 흐름 + 표준 | L3. 디자인 오너십 |
|---|---|---|---|
| 대상 | 사내 어드민 저사용, 내부 실험 | 어드민 고사용, 고객단 저트래픽, 파트너 툴 | 고객단 고트래픽, 돈/신뢰 관련, 새 패턴 선례 |
| 디자이너 역할 | 리뷰어 (코멘트만) | 리뷰어 + 수정 제안 | 오너 (설계에 직접 참여) |
| 타임박스 | 30분 내 비동기 코멘트 | 1시간 내외, 필요 시 짧은 싱크 | 기존 디자인 프로세스 그대로 |
| 보는 것 | 판단 기준 1~4 (안전선) | 판단 기준 1~9 (+ 플로우, 상태/엣지케이스, 디자인 시스템, 핵심 카피) | 판단 기준 1~12 전부 (+ IA, 페르소나/VOC 검증) |
| 안 보는 것 | 픽셀, 컬러 톤, 마이크로카피, 레이아웃 취향 | 픽셀 단위 디테일, 비주얼 폴리시 | 없음 |

L3 판정이면 리뷰로 처리하지 않는다. 프로토타입이 이미 있어도 디자이너가 협업자로 들어간다.

## 3. 오디언스별 리뷰 관점

레벨은 깊이를 정하고, 오디언스는 같은 레벨 안에서 무엇에 힘줄지를 정한다.

| 오디언스 | 특성 | 레벨 판정 시 주의 | 힘줄 것 | 힘 뺄 것 |
|---|---|---|---|---|
| 일반 유저 | 비전문, 대체 서비스로 이탈 쉬움 | 트래픽 기준 그대로 적용 | 첫인상, 이해 용이성, 브랜드 톤, 감성 품질 | (L3면 없음) |
| 셀러/파트너 | 전문 반복 사용자, 돈 직결 (정산·광고비·수수료) | **트래픽 낮아도 돈·데이터 걸리면 한 레벨 올림** | 정보 정확성, 반복 작업 효율, 오류·실수 방지 | 비주얼 폴리시, 감성 품질 |
| 브랜드사 | 소수 담당자, 관계 임팩트 큼, 온보딩 가능 | 사용 빈도보다 관계 리스크로 판정 | 신뢰감·전문성 인상, 데이터 정확성 | 마이크로 인터랙션 |
| 사내 어드민 | 내부, 교육으로 커버 가능 | 기존 기준 그대로 | 안전선, 비가역 액션 | 그 외 전부 (L1 기준) |

## 4. 판단 기준 12개 (피드백의 근거)

**모든 피드백 코멘트는 아래 기준 번호를 인용한다. 기준을 인용할 수 없으면 취향이므로 [사소]로 강등하거나 뺀다.**

코멘트 형식: `[태그] {위치}: {문제} (기준 n {기준명}). 제안: {대안}`

### 안전선 = 기준 1~4 (전 레벨, 위반 시 자동 [블로커])

| # | 기준 | 체크 질문 |
|---|---|---|
| 1 | 오해 방지 | 유저가 라벨·문구를 보고 실제와 다른 결과를 기대하게 되지 않는가 |
| 2 | 비가역 보호 | 되돌릴 수 없는 액션(삭제·발송·확정) 앞에 확인 또는 복구 장치가 있는가 |
| 3 | 정보 신뢰 | 금액·날짜·수치가 정확한가. 불확실한 값이 확정처럼 보이지 않는가 |
| 4 | 패턴 재사용 | 같은 문제를 푸는 기존 컴포넌트·패턴이 있는데 새로 만들지 않았는가 |

### L2 추가 = 기준 5~9 (기본 [제안], 유저 피해가 크면 [블로커])

| # | 기준 | 체크 질문 |
|---|---|---|
| 5 | 과업 완결 | 진입부터 완료까지 막힘 없이 가는가. 완료 후 다음 행동이 이어지는가. 취소·뒤로가기 경로가 있는가 |
| 6 | 상태 완결 | empty·로딩·에러·권한 없음·극단값(0건, 999건, 긴 텍스트)이 정의됐는가 |
| 7 | 일관성 | ODS 컴포넌트를 쓰는가. 같은 동작이 서비스 다른 곳과 같은 방식으로 표현되는가 |
| 8 | 실수 방지·효율 | 자주 하는 작업의 단계 수가 최소인가. 실수하기 쉬운 입력에 가드(기본값·포맷·검증)가 있는가 |
| 9 | 카피 명확성 | 버튼이 결과를 말하는가("확인" 대신 "삭제하기"). 용어가 서비스 표준 용어인가. **판단 근거와 대안 작성 = 오늘의집 UX Writing 가이드 (writing-bot 스킬)** |

### L3 추가 = 기준 10~12 (design-review 풀 워크플로우와 연결)

| # | 기준 | 체크 질문 |
|---|---|---|
| 10 | 유저 가치 적합 | 타겟 페르소나의 목표와 화면의 우선순위가 일치하는가 (페르소나·VOC로 검증) |
| 11 | 첫인상·감성 품질 | 위계·밀도·브랜드 톤이 첫 화면에서 신뢰를 만드는가 |
| 12 | 인터랙션 디테일 | 전환·피드백·마이크로카피가 행동을 자연스럽게 잇는가 |

### 오디언스별 우선 순회 기준

| 오디언스 | 우선 | 제외·후순위 |
|---|---|---|
| 일반 유저 | 9, 11 가중 | 없음 |
| 셀러/파트너 | 3, 8 우선 | 11 제외 |
| 브랜드사 | 3, 11(신뢰 관점) | 12 후순위 |
| 사내 어드민 | 1~4 + 8 | 나머지 후순위 |

## 5. 운영 규칙

### 피드백 태그
모든 코멘트에 셋 중 하나를 단다.

| 태그 | 의미 |
|---|---|
| `[블로커]` | 반영 안 하면 내보내면 안 됨 |
| `[제안]` | 반영 권장, 판단은 PO |
| `[사소]` | 참고만 |

### 리뷰 요청 시 요청자가 적을 3줄
1. 누가 쓰나
2. 얼마나 쓰나
3. 잘못되면 뭐가 깨지나

이 3줄로 레벨을 판정하고, 레벨 협상도 이 기준으로 한다.
