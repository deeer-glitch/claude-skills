# 디자인 피드백 레벨 기준 (v1, 개인용)

> 2026-07-22 작성. PO가 만든 프로토타입에 디자인 피드백을 줄 때, 어디서부터 어디까지 볼지 판정하는 기준.
> 일단 디어 개인 기준으로 운영. 팀 공통 규칙으로 확장 시 "L3 = 리뷰가 아니라 협업 전환" 부분을 PO 조직과 합의 필요.

---

## 1. 판정: 4가지 리스크 질문

레벨을 정하기 전에 아래 4개를 확인한다. 고객단/어드민, 트래픽은 대리 지표일 뿐이고, 실제 기준은 리스크.

| 축 | 질문 | 높음의 예 |
|---|---|---|
| 노출 | 누가 보는가? | 고객 > 파트너/판매자 > 사내 어드민 |
| 규모 | 얼마나 많이, 자주 쓰는가? | 홈·PDP급 트래픽, 매일 쓰는 운영 툴 |
| 되돌리기 비용 | 잘못되면 무슨 일이 생기고, 고치기 쉬운가? | 결제·할인·정산·데이터 삭제, 새 패턴 선례 |
| 수명 | 일회성인가 계속 남는가? | 지속 기능 > 한시 이벤트 > 내부 실험 |

주의: 어드민이라도 새 인터랙션 패턴을 처음 만드는 화면이면 이후 어드민 전체의 선례가 되므로 레벨을 올린다.

축의 역할 분리: 위 4축은 **얼마나 깊게 볼지(레벨)** 를 정하고, 오디언스(§3)는 그 레벨 안에서 **무엇에 힘줄지(관점)** 를 정한다.

## 2. 피드백 3레벨

| | L1. 안전선 체크 | L2. 흐름 + 표준 | L3. 디자인 오너십 |
|---|---|---|---|
| 대상 | 사내 어드민 저사용, 내부 실험 | 어드민 고사용, 고객단 저트래픽, 파트너 툴 | 고객단 고트래픽, 돈/신뢰 관련, 새 패턴 선례 |
| 디자이너 역할 | 리뷰어 (코멘트만) | 리뷰어 + 수정 제안 | 오너 (설계에 직접 참여) |
| 타임박스 | 30분 내 비동기 코멘트 | 1시간 내외, 필요 시 짧은 싱크 | 기존 디자인 프로세스 그대로 |
| 보는 것 | 아래 "안전선"만 | + 플로우 정합성, 상태/엣지케이스(empty·에러·로딩), 디자인 시스템 컴포넌트 준수, 핵심 카피 | + IA, 인터랙션 디테일, 비주얼 완성도, 페르소나/VOC 검증 |
| 안 보는 것 | 픽셀, 컬러 톤, 마이크로카피, 레이아웃 취향 | 픽셀 단위 디테일, 비주얼 폴리시 | 없음 |

L3 판정이면 리뷰로 처리하지 않는다. 프로토타입이 이미 있어도 디자이너가 협업자로 들어간다.

## 3. 오디언스별 리뷰 관점

레벨은 깊이를 정하고, 오디언스는 같은 레벨 안에서 무엇에 힘줄지를 정한다.

| 오디언스 | 특성 | 레벨 판정 시 주의 | 힘줄 것 | 힘 뺄 것 |
|---|---|---|---|---|
| 일반 유저 | 비전문, 대체 서비스로 이탈 쉬움 | 트래픽 기준 그대로 적용 | 첫인상, 이해 용이성, 브랜드 톤, 감성 품질 | (L3면 없음) |
| 셀러/파트너 | 전문 반복 사용자, 돈 직결 (정산·광고비·수수료) | **트래픽 낮아도 돈·데이터 걸리면 한 레벨 올림** | 정보 정확성, 반복 작업 효율, 오류·실수 방지 | 비주얼 폴리시, 감성 품질 |
| 브랜드사 | 소수 담당자, 관계 임팩트 큼, 온보딩 가능 | 사용 빈도보다 관계 리스크로 판정 | 신뢰감·전문성 인상, 데이터 정확성 | 마이크로 인터랙션 |
| 사내 어드민 | 내부, 교육으로 커버 가능 | 기존 기준 그대로 | 안전선, 비가역 액션 | 그 외 전부 (L1 기준) |

## 4. 레벨 무관 안전선 (L1이 보는 전부)

- 유저가 의도와 다른 행동을 하게 되는 구조 (오해되는 버튼, 반대로 읽히는 문구)
- 되돌릴 수 없는 액션에 확인 장치 없음 (삭제, 발송, 정산 확정)
- 잘못된 정보가 사실처럼 보임 (금액, 할인율, 날짜)
- 기존 컴포넌트가 있는데 새로 만듦 (재사용 권고)

### 리뷰 요청 시 요청자가 적을 3줄
1. 누가 쓰나
2. 얼마나 쓰나
3. 잘못되면 뭐가 깨지나

이 3줄로 레벨을 판정하고, 레벨 협상도 이 기준으로 한다.
