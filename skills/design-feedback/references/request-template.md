# 디자인 피드백 요청 양식 (요청자 공유용)

디어가 슬랙 채널이나 개인 DM 안내에 붙여 쓰는 템플릿. 요청자(PO·개발자·PM 등)가 이 양식으로 요청하면 판정에 필요한 정보(오디언스·규모·피해·산출물 성격)가 바로 확보된다.

각 답이 트리거하는 것: 2~4번 → 리뷰 깊이(L1~L3), 2번 → 관점(오디언스), 5번 → 검수 범위(논의용 = 핵심 가이드만 / 산출물형 = 오늘의집 디자인 가이드 전체 + ODS 정합).

---

## 슬랙 공유용 스니펫

```
프로토타입 디자인 피드백이 필요하시면 아래 5줄과 함께 보내주세요!

1. 링크: (Figma나 프로토타입 링크)
2. 누가 쓰나요?: (일반 유저 / 셀러·파트너 / 브랜드사 / 사내 어드민)
3. 몇 명이, 얼마나 자주 쓰나요?: (예: 운영 담당 3명이 주 1회 / 셀러 수천 명이 매일 / 전체 고객 트래픽)
4. 오류나 실수가 나면 무엇이 잘못되나요?: 해당하는 것 모두 골라주세요 (돈이 틀어짐: 결제·정산·할인 / 데이터가 지워지거나 바뀜 / 잘못된 정보가 고객에게 보임 / 없음: 조회용)
5. 이 화면, 어느 단계인가요?: (논의용: 방향 잡는 중이라 가볍게 봐주면 됨 / 산출물형: 이 파일이 거의 그대로 발행·개발로 감)

2~4번으로 얼마나 깊게 볼지 정해요. 5번이 산출물형이면 오늘의집 디자인 가이드 전체랑 디자인 시스템(ODS) 기준까지 대조해서 봐요.
회신은 P0(발행 전 필수)~P3(백로그) 우선순위로 나눠 드리고, 반영하실 수 있게 붙여넣기용 수정 프롬프트나 수정 적용본을 같이 드려요.
```

## 회신 시 우선순위 안내 (처음 요청한 사람에게 한 번만)

```
피드백은 우선순위 뱃지로 나눠 드려요.
- P0: 발행 전 꼭 반영 (이게 0건이면 "내보내도 됨"이라고 알려드려요)
- P1: 중요, 반영 강권장
- P2: 권장, 판단은 요청자 몫
- P3: 백로그. 참고만 하셔도 돼요
```

## 운영 메모

- 양식 없이 온 요청도 거절하지 않는다. 이번 건은 그대로 처리하고, 회신 끝에 "다음엔 이 5줄과 같이 주시면 더 빨라요"로 양식 안내
- 5줄 중 빠진 정보는 디어/Claude가 추론으로 채우되 회신에 전제를 명시 ("어드민 저사용, 논의용 기준으로 봤어요. 아니면 알려주세요")
- 5번 무응답 시 기본값: 화면이 완성도 높으면 산출물형으로 간주하고 확인 질문 1개 포함 (과소 검수보다 과잉 검수가 안전)
