# Data Modes — 더미 / 실측 / 캡처 인용

와이어에 들어갈 데이터(가격·상품명·로고·수치 등) 출처 모드. footnote에 자동 표시.

## 1. 더미 (Dummy) — 기본 모드

**언제**
- PD가 모드 명시 안 한 경우
- *합의용 시안* — 구조·정보 위계만 보면 되는 경우

**원칙**
- 가격: `89,000원` `259,000원` 같은 보편적 둥근 수
- 상품명: "호텔 침구 베딩 세트" "월넛 침대 프레임 Q" 등 일반적이고 그럴듯한 이름
- 브랜드: 실제 오늘의집 입점 브랜드 중 공개 정보 수준 — OK
- 로고: 회색 원/사각 박스 (`.brand-logo`)
- 수치: 5개, 19개 같은 보편 수

**footnote 자동 추가**
```
상품·가격·할인율·로고는 더미 데이터입니다.
```

## 2. 실측 (Real Data)

**언제**
- PD가 *"실측 데이터로 보여줘"*
- *PO·DA와 합의 시안* — 실제 데이터 기반으로 보여줘야 의사결정 가능한 경우
- 캡처 인용 가격·상품을 그대로 넣고 싶을 때

**원칙**
- 캡처 본 실제 가격·상품명·브랜드 인용
- KV 사회적 증거: 실제 `so******님 패키지 5,316,500원` 같은 형태 가능
- 실측 지표 (`user CTR 7.15%` 등) 본문에 노출 가능

**footnote 자동 추가**
```
실측 데이터 인용 — 공유 시 자동 마스킹됩니다.
```

**공유 모드 발동 시 강제**
- 자동 마스킹이 *반드시* 작동. `masking-patterns.json`의 `real_prices_from_capture` 카테고리 강화.
- PD 컨펌 단계에서 *"실측 데이터 들어있어요. 마스킹 잘 됐는지 한 번 더 확인 부탁드려요"* 추가 안내.

## 3. 캡처 인용 (Capture Reference)

**언제**
- PD가 *Figma 캡처·기존 디자인 이미지*를 첨부했고, 와이어 안에 *참고용 이미지로 박고 싶을* 때

**원칙**
- 와이어 안에 캡처 이미지 슬롯 (`.capture-slot`) 삽입
- 캡처 위·아래에 *"참고: 기존 디자인"* 같은 라벨
- 캡처 옆/주변에 *더미 카피로 보강* (직접 인용 X)

**footnote 자동 추가**
```
참고 이미지 인용 — 외부 공유 시 캡처 제거 권장.
```

**공유 모드 발동 시**
- 캡처 이미지가 사내 디자인 자산일 가능성 → PD한테 *"캡처 그대로 공유해도 OK인가요?"* 확인 받음
- OK 안 받으면 캡처 부분만 회색 박스로 치환 (`[참고 이미지 — 공유본에서는 제외]`)

## 모드 선택 추론

| 발화 | 추론 모드 |
|---|---|
| "와이어 만들어줘" (그냥) | 더미 |
| "실측 데이터로", "실제 캡처로" | 실측 |
| 메시지에 캡처 이미지 첨부 + "이거 반영" | 캡처 인용 |
| "더미로 가볍게" | 더미 |

→ 모호하면 한 줄 확인.
