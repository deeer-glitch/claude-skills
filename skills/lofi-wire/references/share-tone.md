# Share Tone — 공유용 톤 변환 룰

공유 모드에서는 *내부 워크플로우 표현*을 평탄화한다. 받는 사람(외부·타 직군)은 PD 작업 컨텍스트를 모르니까 *내부 라벨*이 낯설거나 어색한 느낌을 줄 수 있어서.

## 절대 금지 (공유 모드)

이런 표현이 와이어에 있으면 자동 제거 또는 평문 치환:

| 금지 표현 | 이유 |
|---|---|
| **"PICK"** / **"DESIGN PICK"** | 내부 워크플로우 라벨. 외부는 의미 모름 |
| **"PD 판단으로 픽 요구"** | 내부 메타 표현 |
| **"PRD v0.2 (작성자명, 날짜)"** 같은 *작성자·일자 출처* | 내부 SSOT 노출 |
| **"v2 → v3 변경 — PRD v0.2 반영"** 같은 *내부 버전 트래킹* | 평탄화 |
| **"디자인 결정 필요"** / **"PD 결정"** | "검토 항목" 등으로 |
| **"미확정"** / **"TBD"** 단독 | "추후 확정" 같은 자연스러운 표현 |
| **"실험키 XPC TBD"** | 제거 (실험키 자체 마스킹 대상) |

## 평탄화 치환표

| 내부 모드 | 공유 모드 |
|---|---|
| 변경 노트: *"v2 → v3 변경 — PRD v0.2(5/28) 반영..."* | *"{날짜} 업데이트"* 정도 |
| 섹션 제목: *"A. 브랜드 커뮤니케이션 3안 비교 (PRD가 PD 판단으로 픽 요구)"* | *"A. 브랜드 커뮤니케이션 3안 비교"* |
| anno 제목: *"갱신된 디자인 결정 항목"* | *"디자인 검토 항목"* |
| pill `PICK` | 제거 (그냥 평문) |
| pill `DESIGN PICK` | 제거 |
| pill `CONFIRMED` / `KEPT` / `DROP` / `NEW` | 제거 또는 평문 ("확정 / 유지 / 제외 / 신규") |
| pill `CF` (Needs confirmation) | 제거 ("추후 확정") |
| pill `DEC` (디자인 결정) | 제거 ("검토 필요") |
| footnote *"v3 작성 근거: PRD v0.2 (작성자, 날짜) 본문 fetch · ..."* | *"참고 자료"* 한 줄 + 마스킹된 출처만 |

## 자동 적용 우선순위

`scripts/mask.py`가 이 룰까지 같이 적용. 우선순위:

1. *민감 정보 마스킹* (이름·URL·실측 수치 — `masking-patterns.json`)
2. *내부 표현 평탄화* (위 표 — `share-tone.md` 룰)
3. *footnote 마스킹 안내* 자동 추가

## 케이스 — 내부 모드 와이어를 공유 모드로 갈 때

원본 (내부):
```html
<div class="change-note">
  <b>v2 → v3 변경</b> — PRD v0.2(5/28) 반영.
  <span class="pill new">CONFIRMED</span> 카루셀 확정 · BUNDLE 단독 ·
  카드 안 <b>"패키지 담기" 버튼 유지</b> + <b>스낵바 + 바로가기</b> 패턴 명시 ·
  <span class="pill new">DESIGN PICK</span> 브랜드 커뮤니케이션 3안(PRD 명시) 비교.
</div>
```

공유 모드 변환:
```html
<div class="change-note">
  <b>2026-05-28 업데이트</b> — 카루셀 레이아웃, BUNDLE 모드 단독 스코프,
  카드 안 "패키지 담기" 버튼 + 스낵바 패턴, 브랜드 커뮤니케이션 3안 비교.
</div>
```

## 시각 스타일 보존 (공유 모드에서도)

- 좌측 컬러 띠 금지 — 동일 유지
- 차분한 그레이스케일 — 동일
- 더미 데이터 명시 — 더 강화 (마스킹 안내 footnote)

## 새 표현 발견 시

작업 중 *내부에서만 통하는 표현*이 새로 발견되면:
1. 이 문서에 추가
2. `mask.py`가 자동 적용하도록 `masking-patterns.json`의 `internal_codenames` 카테고리에 정규식 추가
