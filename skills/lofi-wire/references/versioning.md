# Versioning — 버전 비교 자동화

새 버전 만들 때 *이전 버전과의 변경 매트릭스를 자동 생성*해 와이어 안에 박는다. 받는 사람이 *뭐가 바뀌었나*를 한눈에.

## 버전 결정 로직

1. 같은 `{project}-{topic}-{kind}` prefix 파일을 `~/Downloads/`에서 검색
2. 가장 높은 버전 + 1
3. 기존 파일이 없으면 *v 없음* (즉 v1은 보통 생략)
4. v2부터는 명시: `{prefix}-v2.html` `{prefix}-v3.html` ...

## DIFF 매트릭스 자동 생성

새 버전 작성 시 *이전 버전 파일*을 함께 읽어 변경 사항을 추출.

### 추출 항목

| 비교 차원 | 추출 방법 |
|---|---|
| **레이아웃** | 모듈/섹션 수·종류 변화 |
| **컴포넌트** | 새 추가 / 제거된 컴포넌트 |
| **스코프** | 다루는 옵션 수·범위 변화 |
| **카피/문구** | 메인 카피 변화 |
| **인터랙션** | 버튼·이벤트·동선 변화 |
| **데이터 표시** | 가격·라벨·뱃지 표시 형태 |
| **엣지 케이스** | 새로 다룬 / 빠진 케이스 |

### 매트릭스 HTML 박는 위치

상단 변경 노트(`.change-note`) 바로 아래 또는 주석 박스(`.anno`) 안:

```html
<div class="anno">
  <h3><span class="n">DIFF</span> v{이전} → v{현재} 변경 매트릭스</h3>
  <table class="diff-table">
    <tr><th>항목</th><th>v{이전}</th><th>v{현재}</th></tr>
    <tr><td>카드 레이아웃</td><td class="was">2그리드</td><td class="now">수평 카루셀</td></tr>
    ...
  </table>
</div>
```

`.diff-table`의 `td.was` (취소선 회색) / `td.now` (강조)는 `assets/component-library.css`에 박혀 있음.

## 수동 변경 메모 입력

자동 추출이 100% 잡지 못함. PD가 *"이건 강조해서 적어줘"* 발화 시 매트릭스에 추가.

## 짧은 변경 노트 (매트릭스보다 가벼움)

v2/v3 같은 작은 변경은 매트릭스 없이 *한 줄 노트*만:

```html
<div class="change-note">
  <b>v3 업데이트</b> — 카드 안 "패키지 담기" 버튼 + 스낵바 패턴 추가, 브랜드 커뮤니케이션 3안 비교.
</div>
```

→ 모드 선택:
- 변경이 **3건 이하** → 변경 노트 한 줄만
- 변경이 **4건 이상** → DIFF 매트릭스 추가

## 공유 모드에서의 DIFF

공유 모드 변환 시:
- *"v2 → v3 변경"* 같은 내부 버전 표기는 *"{날짜} 업데이트"*로
- 매트릭스 자체는 유지 (받는 사람도 변경을 알아야 함)
- *내부 메타* (PRD 인용·작성자) 는 마스킹
