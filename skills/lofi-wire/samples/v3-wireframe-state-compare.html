관점 OPT 1 칩 + 단일 카루셀 OPT 2 로고 카루셀 OPT 3 추천 모듈 모듈 N행
스코프 위계 한 카루셀 + 칩 분기 — 가장 컴팩트 로고 진입 → 상품 카루셀 (2단) 브랜드 = 1행, N행 누적 — 가장 길어짐
브랜드 인지 텍스트 칩 (약함) 로고 시각화 (강함) 제목 + 작은 로고 (중간)
탐색 효율 한 화면에서 칩 탭만으로 전환 (빠름) 로고 → 카루셀 한 번 더 (중간) 스크롤로 자연스럽게 (느림, 우연 발견)
"기존 UI 재활용"
(PO/DA 요구)		 개인화 카루셀 컴포넌트 그대로 재활용 (✅) 브랜드 묶음 할인 로고 카루셀 컴포넌트 (✅ — 다만 활성도 약함) 추천 모듈 모듈 컴포넌트 그대로 재활용 (✅)
[실험키] 데이터 정합 상품카드 노출 형태 — 데이터 우호적 로고 카루셀 단독은 user CTR +N%로 약했음 — 약점 추천 모듈 모듈 user CTR +N% (1위) — 강점
제목 → 브랜드 묶음 할인 상세 진입
(U-3 P2)		 제목 영역 없음 → 진입로 별도 필요 로고 탭 → 진입로 만들 수 있음 clickable 제목으로 자연스럽게 (✅)
세로 길이 부담 낮음 중간 높음 (N행)
  • 데이터 정합으로는 OPT 3 우세 — XPC1901에서 추천 모듈 모듈이 user CTR 4배 우세.
  • 탐색 효율로는 OPT 1 우세 — 온보딩처럼 빠르게 훑는 맥락에 적합.
  • 브랜드 단독 인지가 중요하면 OPT 2 — 다만 로고 카루셀 단독은 데이터상 약했음.
  • 하이브리드 가능: OPT 3의 모듈 헤더에 OPT 2의 로고를 작게 얹는 형태(=v2 시안) — 두 패턴 결합.

B1 "패키지 담기" 동선

  • 담고 현재 화면 유지 + 스낵바 노출.
  • 스낵바 카피: "패키지에 상품을 추가했어요" + "바로가기" CTA.
  • "바로가기" 탭 → 패키지빌더로 이동 (수동 진입 경로 유지).

B2 디자인 검토 항목

  • 스낵바 안 아이콘 처리(체크 등 사용 여부).
  • "바로가기" 강조 색상.
  • 이미 담긴 상품 카드 상태 표현 (재탭 방지 / 재담기 허용 여부).

C. 정책 및 참고

C1 PRD v0.2 주요 정책

  • 카드 레이아웃 = 카루셀.
  • 스코프 = BUNDLE 모드 단독 (모드 B·C 후속 검토).
  • "패키지 담기" 동작 = 현재 화면 유지 + 스낵바.
  • 카드 구성 = "개인화 카루셀 구성과 동일" (이미지 + 브랜드 + 상품명 + 가격 + 패키지 담기).
  • 브랜드 정렬 = "기존 추천 모듈 로직 그대로 따름".
  • User Stories U-1(P0 상품 카드) / U-2(P1 브랜드 묶음 할인 로고) / U-3(P2 상세 페이지 이동).

C2 온보딩 트리거 조건 수정 (PRD 신규 섹션)

  • 변경: 위시리스트 version 기반 → "패키지 만들기" CTA 클릭 이력 기반.
  • has_completed_onboarding = true 조건:
    • "패키지 만들기" 버튼(go_mypackage) 클릭 시
    • 상품 카드 클릭 시 ← 우리 섹션
    • "패키지 담기" 클릭 시 ← 우리 섹션
    • 그 외 "상품 모듈 관련 영역 클릭" 시 디자인 확정 후 검토
  • 함의: 본 섹션 클릭이 온보딩 완료 신호로 카운트됨. 패키지 페이지 진입을 대체하지 않고 *추가 완료 경로*가 됨.

C3 디자인 검토 항목

  • 브랜드 커뮤니케이션 옵션 1 / 2 / 3 중 선택 (A 비교 매트릭스 참고).
  • 스낵바 토큰·duration·아이콘·바로가기 강조 (B2 참고).
  • 이미 담긴 상품 카드 상태 표현 (재탭 방지 / 재담기 허용).
  • 옵션 3 채택 시 모듈 헤더에 작은 로고 노출 여부.
  • 옵션 1 채택 시 칩에서 브랜드 묶음 할인 상세 진입로 별도 필요.
  • 모듈·카루셀 행 수 상한, 브랜드당 노출 가능 상품 최소 수 (개발 협의).

C4 패키지할인 뱃지 (U-2 / P1)

  • PRD에서 P1로 명시: 패키지할인 대상 상품 카드에 패키지할인 뱃지 노출.
  • Figma 참고: node-id=13283-28355.
  • 옵션 결정 후 카드 디테일에 뱃지 위치 적용.

C5 엣지 케이스 (PRD E1·E4·E5·E6 유지)

  • E-1 노출 가능 브랜드 0 → 섹션 자체 미노출.
  • E-4 담기 불가(품절·단종) → 기존 패키지 담기 에러 동선.
  • E-5 PDP 진입 시 품절·판매중지 → PDP 기본 동작.
  • E-6 딥링크/외부 유입 → 동일 bp-cid 분기.
