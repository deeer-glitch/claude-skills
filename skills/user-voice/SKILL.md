# 유저 제안 대시보드 (유대시) 스킬

## 개요

유저들이 오늘의집 앱 내 '개선 제안' 기능을 통해 직접 남긴 의견 데이터베이스입니다.
기획/디자인 결정의 정성적 근거로 사용할 수 있습니다.

**데이터 출처**: [유대시: 유저 제안 대시보드](https://docs.google.com/spreadsheets/d/1VZl77JKS-Zw_muZJTk1EjENNOuBPL_J-xepg_kS96z8/edit)

---

## 언제 사용하나요

- PRD / 1pager의 "유저 Pain Point" 또는 "정성 데이터" 섹션 작성 시
- 특정 기능(검색, 콘텐츠 탭, 추천, 배송, 리뷰 등)에 대한 유저 불만 파악 시
- 기획 결정에 VOC(Voice of Customer) 근거가 필요할 때
- UX 리서치 가설 수립 전 사전 탐색 시
- 팀 설득 자료에 실제 유저 인용문이 필요할 때

---

## 데이터 구조

### 키워드 분류 체계

대분류(트랙/TF) → 중분류 → 소분류 3단계로 구성됩니다.

**Content TF**
- 콘텐츠: 업로드, 제작/편집, 콘텐츠 타입, 챌린지, 태그
- 크리에이터: 수익화 프로그램, 소통
- 뷰어: 상품/콘텐츠 발견, 상품 탐색, 콘텐츠 탐색, 저장/스크랩

**Commerce TF**
- 커머스 기획: 이벤트/기획전
- 커머스 서비스: 리뷰, 상품 카테고리, 가격, 브랜드, 나만의패키지
- 커머스 플랫폼: 쿠폰, 혜택, 주문/결제, 배송, 교환/환불/클레임, 포인트
- 브랜드사업: PB, 단독

**Search & Rec TF**
- 추천, 검색(기능), 검색

**기타**
- 사용성: OS, 신뢰도, 심미성
- 커뮤니티: 커뮤니티홈
- O2O: 오프라인 관련
- 소통: 앱 내 소통 기능

## 사용 방법

### 1단계: 키워드 파악

조회하려는 기능/영역의 키워드를 아래 분류 체계에서 찾으세요.

예시:
- 콘텐츠 탭 몰입 개선 → `콘텐츠 탐색`, `콘텐츠 타입`, `추천`, `검색(기능)`
- 검색 개편 → `검색(기능)`, `검색`, `상품 탐색`
- 리뷰 기능 → `리뷰`, `신뢰도`

### 2단계: Google Sheets에서 직접 데이터 조회

**스프레드시트 ID**: `1VZl77JKS-Zw_muZJTk1EjENNOuBPL_J-xepg_kS96z8`

`mcp__google-workspace-mcp__read_sheet_values` 도구를 사용하여 직접 데이터를 읽어옵니다.

**조회 전략:**

1. **최신 의견 조회** — raw 시트는 최신순 정렬 (Row 2가 가장 최신)
   - 최근 500건: `raw!A1:C501`
   - 최근 1000건: `raw!A1:C1001`
   - 필요 시 더 넓은 범위 조회 가능

2. **키워드 분류 체계 확인** — 관련 키워드 파악 시
   - 범위: `1. 키워드 분류 테이블!B3:D100`
   - 대분류(트랙/TF) → 중분류 → 소분류 구조

3. **키워드 빈도 확인** — 트렌드 파악 시
   - 범위: `2. 키워드 카운트!A1:U100`

**조회 방법:**
```
read_sheet_values(
  spreadsheet_id: "1VZl77JKS-Zw_muZJTk1EjENNOuBPL_J-xepg_kS96z8",
  range_name: "raw!A1:C1001"
)
```

**데이터 필터링:**
- raw 시트에서 읽어온 데이터 중 '개선 의견' 컬럼(C열)에서 관련 키워드를 텍스트 매칭으로 필터링
- 여러 키워드를 OR 조건으로 검색하여 관련 의견을 최대한 수집
- 날짜 범위가 필요하면 '등록일 추출' 컬럼(B열) 기준으로 필터링

## 콘텐츠탭 관련 주요 유저 의견 (요약)

콘텐츠탭 관련 작업 시 참고할 주요 VOC 패턴:

**발견/탐색 불편**
- 원하는 스타일/취향 콘텐츠 찾기 어려움
- 추천이 반복적이거나 비슷한 콘텐츠만 나옴
- 인스타그램처럼 취향 기반 피드 구성 요청

**콘텐츠 타입 구분**
- 협찬/광고 콘텐츠와 실제 후기 구분이 어려움
- 포토리뷰, 집들이, 노하우 등 타입별 분리 탐색 요청

**저장/스크랩**
- 스크랩 후 다시 찾기 어려움
- 위시리스트, 폴더 기능 요청

## 주의사항

- 데이터는 **앱 내 제안 기능**을 통해 수집된 것으로, 적극적 의견 표출 유저 편향 있음
- 침묵하는 다수(silent majority)의 의견은 별도 리서치로 보완 필요
- 정량 데이터(클릭률, 전환율 등)와 함께 삼각검증하여 활용 권장
- 최신 데이터 확인은 스프레드시트 직접 접근 권장
