# Spec-policy-handoff 스킬

Figma 화면 구조를 읽고 개발 핸드오프용 주석 초안을 생성하는 팀 공용 스킬.

---

## Step 0 — 첫 사용 여부 확인

아래 신호를 기반으로 판단하며, 물어보는 것은 최후 수단이다.

**가이드 없이 바로 Step 1 진행하는 경우**
- URL 또는 이미지를 바로 전달한 경우 → 사용법을 아는 사람이므로 바로 진행
- "다시", "이번엔", "이 화면도" 등 연속 작업 표현이 있는 경우 → 바로 진행

**가이드를 먼저 안내하는 경우**
- "처음이에요", "어떻게 써요", "사용법 알려줘" 등 명시적 표현 → 바로 안내
- "주석 써줘", "핸드오프 주석"처럼 URL/이미지 없이 스킬만 언급한 경우 → 아래 문장으로 한 번 확인

```
처음 사용하시나요? 처음이시라면 스킬 사용 가이드를 먼저 안내해드릴게요.
```

가이드가 필요한 경우 `references/user-guide.md`를 읽어서 안내한 뒤 Step 1로 진행.

---

## Step 1 — 인풋 확인 및 처리 방식 분기

Figma URL 또는 이미지와 함께 아래 맥락 정보가 있으면 같이 받는다.
없으면 Figma 구조만으로 진행하되, [⚠️ 입력 필요] 항목이 늘어날 수 있음을 안내한다.

```
함께 주시면 주석 품질이 올라가요 (선택)
• 이 화면의 핵심 정책이나 결정사항
• PRD / 1Pager 요약
• 이전 대화에서 확정된 내용
```

인풋 종류에 따라 아래와 같이 분기한다.

### A. 단일 프레임 URL
`get_design_context` 1회 실행 → Step 2로 바로 진행.

### B. 섹션 URL
1. `get_metadata`로 섹션 구조 읽기
2. 섹션 내 주요 화면 프레임 목록을 사용자에게 표시:

```
이 섹션에 주석이 필요한 화면이 N개 있습니다.

1. [프레임명] — node-id: XXXX:XXXX
2. [프레임명] — node-id: XXXX:XXXX
...

전체 처리할까요, 특정 화면만 할까요?
전체면 "전체", 특정 화면이면 번호를 말씀해주세요.
```

3. 응답에 따라:
   - "전체": 프레임 2~3개씩 묶어 순차적으로 `get_design_context` → 주석 초안 각각 생성
   - 번호 지정: 해당 프레임만 `get_design_context` → 주석 초안 생성

**프레임 목록 추출 기준**
- 섹션 직속 FRAME 노드만 포함 (width 375 내외, 실제 화면 단위)
- Annotation/Guide/Design Guide 섹션, 해상도 대응 프레임 등 보조 자료는 제외
- CMS 화면 등 별도 도메인 프레임은 "[CMS]" 등 접두어로 구분하여 포함

### C. 이미지 인풋
이미지를 직접 분석해서 화면 구조 추론 후 주석 초안 생성.
- lokalize key, data-spec-annotations, hidden 상태 프레임은 확인 불가 → `[⚠️ 입력 필요]` 처리
- Figma URL도 함께 주어지면 A/B 방식으로 전환

### D. URL/이미지 없이 텍스트만
기능명과 화면 구조를 질문하여 수동으로 채움.

## Step 2 — 주석 초안 생성 및 출력 (항상 실행)

각 화면별로 아래 포맷에 맞게 초안 생성 후 출력.
섹션 전체 처리 시 화면별로 구분선(`---`)을 두어 순서대로 출력.

출력 형식:
```
📋 Figma 주석 초안 — [프레임명]
(아래 텍스트를 복사해서 Figma 주석에 붙여넣으세요)

{생성된 주석 텍스트}

---
[⚠️ 입력 필요] 표기된 항목은 직접 채워주세요.
(추론) 표기된 항목은 내용을 확인 후 수정해주세요.
```

복수 화면인 경우 모든 초안 출력 후 한 번만 위 안내 표시.

---

## Step 3 — Figma 직접 작성 안내 (항상 실행)

초안 출력 직후 아래 문구를 항상 붙인다.

```
Figma에 직접 달아드릴 수도 있어요.
원하시면 주석을 달 프레임/컴포넌트를 Figma에서 선택한 뒤 "Figma에 달아줘"라고 해주세요.
(단, 섹션 내 인스턴스는 직접 접근이 어려워 소스 컴포넌트에 달릴 수 있습니다)
```

---

## Step 4 — Figma 직접 작성 실행 (선택, 요청 시에만)

"Figma에 달아줘", "직접 달아줘" 등 명시적 요청이 있을 때만 실행.

```javascript
var selection = figma.currentPage.selection;
if (selection.length === 0) {
  figma.closePlugin("선택된 노드가 없습니다. Figma에서 주석을 달 프레임/컴포넌트를 선택해주세요.");
  return;
}

var node = selection[0];
var specText = "SPEC_TEXT";

if ("annotations" in node) {
  node.annotations = [{ label: specText, properties: [] }];
  figma.closePlugin("주석 작성 완료: " + node.name + " (" + node.type + ")");
} else if (node.type === "GROUP") {
  var parent = node.parent;
  while (parent && !("annotations" in parent)) { parent = parent.parent; }
  if (parent) {
    parent.annotations = [{ label: specText, properties: [] }];
    figma.closePlugin("GROUP 미지원 → 부모 [" + parent.name + "]에 작성 완료");
  } else {
    figma.closePlugin("작성 가능한 부모 노드 없음. 텍스트를 수동으로 붙여넣어주세요.");
  }
} else {
  figma.closePlugin("이 노드 타입은 주석 미지원: " + node.type);
}
```

섹션 내 인스턴스 등 접근 불가 시 → 소스 컴포넌트에 작성 후 안내:
"섹션 내 인스턴스는 직접 접근이 어려워 소스 컴포넌트 [컴포넌트명]에 작성했습니다."

## 주석 포맷

Figma 네이티브 주석에 붙여넣는 텍스트.
마크다운 헤딩(#) 없이 일반 텍스트 + 불릿으로만 구성.

```
{기능명}
{한 줄 설명 — 언제, 왜 이 화면이 노출되는가}

공통 스펙
• 위치: {피드/탭/모달 등 노출 위치}
• {개발 우선순위 — Figma에 명시된 경우만}

1. 노출 조건
• {언제 이 화면/컴포넌트가 노출되는가}

2. {주요 구성 요소 또는 동작명}
• {설명}

3. {상태 정의 — Loading / Empty / Error 등, 해당 시}
• Loading: {트리거 조건 + UI 처리}
• Empty: {트리거 조건 + UI 처리} / 문구: {lokalize key}
• Error: {트리거 조건 + UI 처리} / 문구: {lokalize key}

4. 페이지 랜딩 인터랙션
• {액션}: {이동 대상 + 상태 변화}
• 페이지 전환 방식: {Stack / Modal / Replace 등}

5. 진입 애니메이션
라이브러리: [⚠️ 입력 필요]
• opacity, duration, easing: [⚠️ 입력 필요]
자식 요소 순차 등장:
• delay, interval: [⚠️ 입력 필요]
```

## 섹션별 작성 규칙

**기능명 / 한 줄 설명**: 프레임명 기반 / "언제, 왜 노출되는가"로 작성

**공통 스펙**: 레이어 계층에서 위치 추론 / 우선순위는 명시된 경우만

**노출 조건**: hidden 프레임, 조건부 컴포넌트에서 추론 → `(추론)` 표기

**구성 요소**: 눈에 보이는 요소 기준으로 섹션 분리 / ODS 컴포넌트는 prefix 제거

**상태 정의**: skeleton/empty/error 별도 프레임 있으면 각각 기술 / data-spec-annotations Phase 정보 반영 / lokalize key 함께 표기

**랜딩 인터랙션**: 탭 가능한 요소 열거 / 전환 방식 명시 / 읽히지 않으면 `[⚠️ 입력 필요]`

**진입 애니메이션**: 항상 `[⚠️ 입력 필요]` / 섹션 삭제 금지

## 노드 타입별 주석 지원 현황

| 타입 | 지원 | 처리 방식 |
|------|------|-----------|
| FRAME | ✅ | 직접 작성 |
| INSTANCE | ✅ | 직접 작성 |
| COMPONENT | ✅ | 직접 작성 |
| COMPONENT_SET | ✅ | 직접 작성 |
| GROUP | ❌ | 부모 FRAME에 작성 후 안내 |
| SECTION / PAGE | ❌ | 작성 불가, 올바른 노드 선택 요청 |

## 운용 원칙

- Step 0(첫 사용 확인)은 스킬 첫 실행 시에만. URL/이미지를 바로 주면 생략하고 Step 1 진행
- Step 2(텍스트 출력) + Step 3(직접 작성 안내)는 항상 실행
- Step 4는 명시적 요청 시에만
- 섹션 전체 처리 시 프레임 목록 먼저 확인 후 사용자 선택 대기
- 전체 처리 시 2~3개씩 묶어 순차 생성 (컨텍스트 과부하 방지)
- Figma에서 읽히지 않는 정보는 `[⚠️ 입력 필요]` / 추론은 `(추론)` 표기
- 팀 전체 스킬이므로 특정 개인 표현 금지
- 노션 저장은 명시적 요청 시에만

## Step 5 — 이어받기 요약 (항상 실행)

주석 초안 출력 완료 후 반드시 아래 형식으로 요약을 출력한다.
다음 대화 시작 시 이 내용을 그대로 붙여넣으면 맥락이 이어진다.

```
[이어받기 요약]
• 작업 화면: {화면명 또는 섹션명}
• 완료한 것: {작성된 주석 화면 목록}
• [⚠️ 입력 필요] 항목: {채워야 할 항목 요약}
• 다음 단계: {보완 후 재작성 필요 여부 또는 다음 작업}
```
