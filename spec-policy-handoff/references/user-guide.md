# Spec-policy-handoff 스킬 사용 가이드

## 기본 사용법

Figma URL 또는 화면 이미지를 주면 주석 초안을 생성해드립니다.

**단일 프레임**: 해당 프레임 주석 1개 생성
**섹션 링크**: 포함된 화면 목록을 먼저 보여드리고, 전체 또는 특정 화면 선택 후 생성
**이미지**: 시각 구조 기반으로 추론하여 생성 ([⚠️ 입력 필요] 항목이 더 많아질 수 있음)

## Figma에 직접 달기 (선택)

초안 출력 후 "Figma에 달아줘"라고 하면 직접 작성해드립니다.
이때 Figma에서 주석을 달 프레임/컴포넌트를 미리 선택해두세요.

## 주석 품질을 높이는 방법

스킬이 자동으로 채우는 영역과 사람이 직접 채워야 하는 영역이 다릅니다.
맥락을 잘 쌓아둘수록 [⚠️ 입력 필요] 항목이 줄어듭니다.

### 스킬이 자동으로 읽는 것
- 레이어 구조, 컴포넌트 목록, 치수/색상
- hidden 처리된 상태 프레임 (skeleton, empty, error)
- `🔑 lokalize key` prefix 레이어명
- `data-spec-annotations` 기존 주석 내용

### 사람이 채워야 하는 것
- Phase별 로딩 순서, 전환 애니메이션 수치
- 페이지 전환 방식 (Stack / Modal / Replace)
- 비즈니스 정책 (P0/P1/P2, 데이터 소스 등)
- API 연동 조건, 권한별 분기

### 맥락 쌓는 방법

**Figma 파일에**
- `data-spec-annotations`로 정책 주석 달기 (MCP가 자동으로 읽음)
- 레이어명에 상태/조건 명시 (`🔑`, `[P0]`, `hidden` 등 일관된 prefix)
- Section/Annotation 프레임으로 정책 시각화

**Claude 대화에**
- 과업 시작 시 PRD나 정책 문서를 대화에 붙여넣기
- 이전 대화에서 결정된 정책을 새 대화 시작 시 요약해서 전달
- Claude Project 시스템 프롬프트에 반복되는 정책 고정

**CLAUDE.md에**
- 자주 쓰이는 정책 패턴, 용어, 컨벤션 정리
- 새 대화에서도 일관된 맥락 유지 가능
