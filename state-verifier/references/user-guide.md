# State-verifier 스킬 사용 가이드

## 목적

디자인 단계에서 UI 상태 누락과 엣지케이스를 선제적으로 점검하여
개발/QA 단계의 이슈를 최소화합니다.

## 기본 사용법

Figma URL 또는 화면 이미지를 주면 아래 두 가지를 분석하여 체크리스트로 출력합니다.

**상태 정의 체크**: Loading, Skeleton, Empty, Error, Success 등 상태 프레임이 존재하는지 확인하고 누락된 상태를 알려줍니다.

**엣지케이스 생성**: 화면 내 컴포넌트를 분석하여 텍스트 최대 길이, 이미지 로드 실패, 빈 목록, 네트워크 오류 등 개발/QA에서 놓치기 쉬운 케이스를 뽑아줍니다.

## 인풋 방식

**단일 프레임 URL**: 해당 화면 1개 분석
**섹션 링크**: 포함된 화면 목록을 먼저 보여주고, 전체 또는 특정 화면 선택 후 생성
**이미지**: 시각 구조 기반으로 추론하여 생성 (상태 프레임 탐지 정확도가 낮을 수 있음)

## 체크리스트 보는 법

**✅ 확인됨**: Figma에서 실제로 확인된 상태 또는 컴포넌트

**⚠️ 누락 의심**: Figma에서 확인되지 않았으나 해당 화면에 필요할 가능성이 있는 상태. 반드시 추가해야 하는 건 아니고, 필요 여부를 검토해보세요.

**□ 엣지케이스**: 개발/QA 전에 디자인 차원에서 확인이 필요한 케이스. 이미 정의된 항목은 체크해두고, 미정의 항목은 주석이나 별도 스펙 문서에 정리해두는 것을 권장합니다.

## 체크리스트 품질을 높이는 방법

Figma 파일에 상태 프레임이 잘 분리되어 있을수록 탐지 정확도가 높아집니다.

- 상태 프레임명에 키워드 포함: `skeleton`, `empty`, `error`, `loading`
- `🌀 Empty` 등 ODS 컴포넌트를 사용한 상태 화면 구성
- lokalize key 레이어명(`🔑 COMMON__ERROR__*`)으로 오류 문구 명시
